# CHANGE REPORTS

Questa cartella contiene mini report sintetici post-task.

## A cosa serve
- tracciare in modo rapido cosa e stato fatto
- mantenere continuita tra interventi diversi
- supportare debug e analisi regressioni
- conservare memoria tecnica operativa del progetto

## Cosa contiene
- un file report per ogni task con modifiche
- template ufficiale `_TEMPLATE_CHANGE_REPORT.md`

## Regole operative
- usare la convenzione nome file definita in `docs/product/CODEX_CHANGE_REPORT_RULES.md`
- compilare solo informazioni verificate dal task
- niente codice completo o diff completi
